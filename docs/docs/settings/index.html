Overview

Logo

/uploads/2018/01/settings-logo.pngThe logo setting allows you to customize the branding of Yakkl CMS by allowing you to replace the Yakkl logo in various touchpoints in the CMS.

Site Name

/uploads/2018/01/settings-sitename.pngSite name allows you to customize the display name of your site in the Yakkl UI. This will update the display in the dashboard, in notifications, and various touchpoints.

URL

/uploads/2018/01/settings-url.pngThe URL setting allows you to provide the URL to your production website, which allows editors to open the production website from the sidebar.

Timezone

/uploads/2018/01/settings-timezone.pngThe timezone setting allows you to configure what timezone should be used when saving dates, previewing, and publishing. This setting effects the output of the date field

Deployment

/uploads/2018/01/settings-deployment.pngThe deployment settings allow you to configure the behaviour of the CMS related to your hosting settings.

Options

  • Deploy on Git Push: when enabled, Yakkl will automatically publish your site when changes are pushed to your Git repository.

Project Paths

/uploads/2018/01/settings-projectpaths.pngProject paths allow you to configure features of Yakkl related to your repository.

Options

  • Admin Path: the path from the root of your repository to where the Remote Admin should be deployed.

File Paths

/uploads/2018/01/settings-filepaths.png

The File Paths settings allow you to configure how the Media Library operates. See the Media Library documentation for more information.

Hugo Version

Hugo Version allows you to configure which Hugo version Yakkl uses to build your Hugo site.

Look here for information on how to configure which version of Jekyll Yakkl uses to build your site.

New File Format

/uploads/2018/01/settings-newfileformat.pngThe new file format setting allows you to configure what file format Yakkl should create new content in.

Options

  • Markdown: will create all new content as .md files. The default and recommended setting.
  • HTML: will create all new content as .html files. This is a beta feature.

This setting affects whether the Markdown Editor or a plaintext editor (for HTML) will be used when creating new content through Yakkl.

Webhook URL

/uploads/2018/01/settings-webhook.pngThe Webhook URL setting allows you to configure an outgoing webhook to trigger third-party applications when events occur in Yakkl. Learn more in the Webhooks documentation.
