      Connecting to a Source Provider

      Yakkl saves changes to your content by committing back to your Git repository. Adding a site to Yakkl first requires that the source code for your site is available through one of our supported source providers. When setting up a new site in Yakkl, you will be prompted to connect to your preferred source provider to link your repository.

      Currently, Yakkl supports the use of Github, GitLab, Bitbucket, and Azure DevOps Repos as source providers.

      Quick Setup via OAuth

      This is the recommeded way to connect your repository. Yakkl will connect to your Git provider and automatically configure your repository to work with Yakkl.

      After selecting your Git provider, ensure the Quick Setup via OAuth toggle is activated and click Next. You will be prompted to authenticate with the selected provider if not already logged in.

      At this time, Quick Setup is only available for Github and GitLab providers.

      Detailed Quick Setup Guides

      • Github
      • GitLab

      Manual Setup

      You may opt to manually setup your source provider by deactivating the Quick Setup via OAuth toggle. This toggle will automatically deactivate when selecting a source provider that does not currently support Quick Setup.

      With manual setup, you will need to specify the repository URL to Yakkl, and add Yakkl’s SSH key and webhook to your source provider.
