Use Yakkl With Azure DevOps Repos

Azure DevOps is a suite of code collaboration and deployment tools provided by Microsoft. Azure DevOps includes cloud hosting for git repositories, similar to Github, GitLab, and Bitbucket. As of today, you can now import Azure DevOps repositories into Yakkl! Yakkl will sync the contents of your Azure DevOps repository by importing changes when they’re pushed to the repo, and committing content updates that you make in Yakkl. Importing Azure DevOps Repos to Yakkl Our OAuth-based Quick Setup is not currently available for Azure DevOps repositories, so you must use our manual setup option when connecting to Azure DevOps.

Gulp and Webpack: Getting the Best of Both Worlds

This makes Gulp a less attractive option for packaging your JavaScript files.

Introducing Custom Build Commands

We’re excited to announce Custom Build Commands! You can now decide what build command should be run for your project. Until now Yakkl chose the build command that needed to be run for you, which worked fine for most cases. However, this meant that projects that needed a custom build process were unable to use Yakkl’s deployment tools and (in some cases) previews. Picking your own commands enables you to really choose your own workflow and give you the flexibility you need.
